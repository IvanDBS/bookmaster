# План рефакторинга MasterDashboard.vue

## Проблема
Файл `frontend/src/views/MasterDashboard.vue` содержит 1500 строк кода и выполняет множество разных функций:
- Управление календарем записей
- Управление услугами
- Управление записями
- Модальные окна
- Навигация

## Решение
Разделение на более мелкие компоненты с четким разделением ответственности.

## Структура после рефакторинга

### Composables (Логика)
```
frontend/src/composables/
├── useCalendar.js      # Логика календаря
├── useBookings.js      # Логика записей
└── useServices.js      # Логика услуг
```

### Компоненты
```
frontend/src/components/master/
├── Calendar/
│   ├── Calendar.vue              # Основной календарь
│   ├── CalendarMonth.vue         # Месяц календаря
│   ├── CalendarLegend.vue        # Легенда календаря
│   └── CalendarSlots.vue         # Слоты выбранного дня
├── Bookings/
│   ├── BookingsList.vue          # Список записей
│   ├── BookingCard.vue           # Карточка записи
│   └── BookingFilters.vue        # Фильтры записей
├── Services/
│   ├── ServicesList.vue          # Список услуг
│   ├── ServiceCard.vue           # Карточка услуги
│   └── ServiceModal.vue          # Модал добавления/редактирования услуги
└── Dashboard/
    └── WelcomeSection.vue        # Приветственная секция
```

## Преимущества рефакторинга

### 1. Разделение ответственности
- Каждый компонент отвечает за одну конкретную функцию
- Легче тестировать и поддерживать
- Упрощается отладка

### 2. Переиспользование кода
- Composables можно использовать в других компонентах
- Логика вынесена из UI компонентов

### 3. Улучшенная читаемость
- Каждый файл содержит меньше кода
- Легче понять структуру приложения
- Проще для новых разработчиков

### 4. Модульность
- Компоненты можно независимо разрабатывать и тестировать
- Легче добавлять новую функциональность

## Совместимость

### API
Все API вызовы остались без изменений, используются те же эндпоинты.

### Состояние
Состояние приложения сохраняется через composables, которые используют те же reactive переменные.

### Навигация
Навигация между страницами работает так же, как и раньше.

### Связанные страницы
- `ScheduleSettings.vue` - не требует изменений
- `AppHeader.vue` - не требует изменений
- `ConfirmationModal.vue` - не требует изменений
